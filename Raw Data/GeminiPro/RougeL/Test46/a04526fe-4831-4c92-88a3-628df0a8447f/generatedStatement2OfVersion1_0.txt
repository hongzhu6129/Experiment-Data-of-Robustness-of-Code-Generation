

**Test Cases:**

**1. Two matching rightmost digits:**

* Input: 12, 22, 33
* Expected Output: true

**2. All three matching rightmost digits:**

* Input: 15, 25, 35
* Expected Output: true

**3. No matching rightmost digits:**

* Input: 11, 22, 33
* Expected Output: false

**4. Zeroes as rightmost digits:**

* Input: 10, 20, 30
* Expected Output: true

**5. Single-digit numbers:**

* Input: 1, 1, 2 
* Expected Output: true

**6.  Large Numbers:**

* Input: 123456789, 987654321, 111111111
* Expected Output: false

**7. Corner Cases (using zero and near maximum integer value):**

* Input: 0, 0, 0
* Expected Output: true

* Input: Integer.MAX_VALUE, Integer.MAX_VALUE -1, Integer.MAX_VALUE - 2 //Integer.MAX_VALUE ends in 7
* Expected Output: false

* Input: Integer.MAX_VALUE, Integer.MAX_VALUE / 10 * 10 + 7,  Integer.MAX_VALUE -1000 + 7
* Expected Output: true  // These numbers are chosen so they have the same last digit as MAX_VALUE


**8. Negative Input:**

* Input: -1, 2, 3
* Expected Output: Invalid input: Please enter non-negative integers.


These test cases cover various scenarios, including matching digits, no matching digits, zeroes, single digits, large numbers, and boundary cases, ensuring comprehensive testing of the code's logic and handling of potential edge cases.  It's important to test with negative input to make sure your input validation works.
