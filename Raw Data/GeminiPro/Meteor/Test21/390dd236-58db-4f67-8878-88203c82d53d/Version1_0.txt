```java
import java.util.Scanner;

public class OctalToBinary {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("Enter an octal number: ");
        String octal = scanner.nextLine();

        try {
            long decimal = 0;
            long power = 1;

            // Octal to Decimal conversion
            for (int i = octal.length() - 1; i >= 0; i--) {
                int digit = octal.charAt(i) - '0';
                if (digit < 0 || digit > 7) {
                    System.out.println("Invalid octal number.");
                    return;
                }
                decimal += digit * power;
                power *= 8;
            }

            // Decimal to Binary conversion
            String binary = "";
            if (decimal == 0) {
                binary = "0"; 
            } else {
                while (decimal > 0) {
                    binary = (decimal % 2) + binary;
                    decimal /= 2;
                }
            }


            System.out.println("Binary equivalent: " + binary);


        } catch (NumberFormatException e) {
            System.out.println("Invalid input. Please enter a valid octal number.");
        }
    }
}
```

**Test Cases:**

1. **Valid Octal Number:**
   - Input: 123
   - Expected Output: 1010011

2. **Zero:**
   - Input: 0
   - Expected Output: 0

3. **Large Octal Number:**
   - Input: 7777777
   - Expected Output: 111111111111111111111

4. **Invalid Octal Number (Containing digits other than 0-7):**
   - Input: 128
   - Expected Output: Invalid octal number.

5. **Invalid Octal Number (Negative Number):**
   - Input: -123
   - Expected Output: Invalid input. Please enter a valid octal number. (The program handles this as a `NumberFormatException` because the initial parsing to long fails).

6. **Invalid Octal Number (Empty String):**
    - Input: ""
    - Expected Output: Invalid input. Please enter a valid octal number.

7. **Invalid Octal Number (Leading Zeroes):**
   - Input: 000123
   - Expected Output: 1010011 (Leading zeroes should be handled correctly)



These test cases cover the basic scenarios, including valid inputs, zero, large numbers, and various types of invalid inputs, to help ensure the program functions correctly under different conditions.  The corner cases are the zero, empty string and the maximum octal input. The error cases are handled robustly.
