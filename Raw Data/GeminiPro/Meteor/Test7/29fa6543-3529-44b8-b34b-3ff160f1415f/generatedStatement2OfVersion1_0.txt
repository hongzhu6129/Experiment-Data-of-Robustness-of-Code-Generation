


## Test Cases

Here are some test cases, including corner cases, to test the `AverageCalculator` program:

**Normal Cases:**

1. **Input:** 10, 20, 30
   **Expected Output:** 20.0

2. **Input:** 2.5, 5.5, 8.0
   **Expected Output:** 5.333333333333333

3. **Input:** -5, 0, 5
   **Expected Output:** 0.0

**Corner Cases:**

4. **Input:** 0, 0, 0
   **Expected Output:** 0.0

5. **Input:** Integer.MAX_VALUE, Integer.MAX_VALUE, Integer.MAX_VALUE  // Test for potential overflow
   **Expected Output:** 2.147483647E9 (or very large number - might vary slightly due to floating-point representation)

6. **Input:** Integer.MIN_VALUE, Integer.MIN_VALUE, Integer.MIN_VALUE // Test for potential underflow
   **Expected Output:** -2.147483648E9 (or very large negative number)


**Error Cases (Input Validation):**

7. **Input:** 10, a, 20   // Non-numeric input
   **Expected Output:** "Invalid input. Please enter numbers only."

8. **Input:** 10, 20   // Insufficient input 
   **Expected Output:**  The program will likely throw an `InputMismatchException` or `NoSuchElementException` because it expects three numbers but only receives two. The `try-catch` block handles the exception gracefully.

9. **Input:**  10 20 30 (with spaces only, no commas or other delimiters)
   **Expected Output:**  This will likely be handled correctly as the `Scanner` defaults to whitespace as a delimiter. The average should be 20.0



**How to run the test cases:**

You can either:

* **Manually:** Run the program multiple times, entering each set of test input.
* **Automated Testing (Recommended):** Use a testing framework like JUnit to write unit tests for the `calculateAverage` method. This is the preferred approach for robust testing.
