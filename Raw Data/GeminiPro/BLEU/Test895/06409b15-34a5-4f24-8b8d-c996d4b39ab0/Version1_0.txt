```java
import java.util.Arrays;

public class RemoveElement {

    public static int removeElement(int[] nums, int val) {
        int k = 0; // Variable to track the length of the new array

        for (int i = 0; i < nums.length; i++) {
            if (nums[i] != val) {
                nums[k] = nums[i];
                k++;
            }
        }

        return k;
    }

    public static void main(String[] args) {
        int[] nums = {1, 4, 6, 7, 6, 2};
        int val = 6;

        System.out.println("Original array: " + Arrays.toString(nums));

        int newLength = removeElement(nums, val);

        System.out.println("The length of the new array is: " + newLength);


        // Print the updated array (optional - the problem only asks for the length)
        System.out.print("Updated array: [");
        for (int i = 0; i < newLength; i++) {
            System.out.print(nums[i]);
            if (i < newLength - 1) {
                System.out.print(", ");
            }
        }
        System.out.println("]");
    }
}
```


Explanation:

1. **`removeElement(int[] nums, int val)` method:**
   - Takes the integer array `nums` and the value to remove `val` as input.
   - Initializes `k` to 0.  `k` will keep track of the index where the next non-`val` element should be placed.
   - Iterates through the array using a `for` loop.
   - Inside the loop, if the current element `nums[i]` is *not* equal to `val`, it copies that element to the `k`-th position and increments `k`. This effectively overwrites the elements that are equal to `val` with elements that are not equal to `val`.
   - Finally, it returns the new length `k`.

2. **`main(String[] args)` method:**
   - Creates an example array `nums`.
   - Sets the value to remove `val`.
   - Prints the original array.
   - Calls the `removeElement` method to modify the array and get the new length.
   - Prints the new length.
   - (Optional) Prints the updated array elements up to the new length.  The original array is modified in-place, so elements beyond `newLength` are not relevant.


Key Improvement:  This version uses a two-pointer approach (the `i` and `k` indices) which is more efficient than creating a new array. It modifies the original array in place.
