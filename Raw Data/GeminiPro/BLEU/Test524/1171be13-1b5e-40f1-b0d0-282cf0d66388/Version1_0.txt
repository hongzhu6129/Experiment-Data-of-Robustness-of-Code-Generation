```java
public class CubeChecker {

    public static void main(String[] args) {
        if (args.length != 1) {
            System.out.println("Please provide exactly one integer argument.");
            return;
        }

        try {
            int num = Integer.parseInt(args[0]);

            if (isCube(num)) {
                System.out.println("number is a cube.");
            } else {
                System.out.println("number is not a cube.");
            }

        } catch (NumberFormatException e) {
            System.out.println("Invalid input. Please enter an integer.");
        }
    }


    private static boolean isCube(int num) {
        // Handle negative numbers.  The cube root of a negative number is negative.
        if (num < 0) {
            return isCube(-num); // Check the absolute value for simplicity
        }

         // Binary search for cube root.  More efficient than brute force
        int low = 0;
        int high = num; 

        while (low <= high) {
            int mid = low + (high - low) / 2; // Avoid potential overflow
            long cube = (long)mid * mid * mid; // Use long to handle large cubes

            if (cube == num) {
                return true;
            } else if (cube < num) {
                low = mid + 1;
            } else {
                high = mid - 1;
            }
        }

        return false;
    }
}
```

**Explanation and Improvements:**

1. **Input Handling:** The code now checks if exactly one argument is provided. It also includes error handling using a `try-catch` block to catch `NumberFormatException` if the input is not a valid integer.

2. **Negative Number Handling:** The `isCube` method now correctly handles negative numbers.  The cube of a negative number is negative.

3. **Efficiency (Binary Search):** The most significant improvement is the use of a binary search algorithm to find the cube root.  This is much more efficient than a simple brute-force loop, especially for larger numbers.  The complexity is O(log n) instead of O(n^(1/3)).

4. **Long for Cube Calculation:** The `cube` variable is now of type `long` to prevent potential overflow when calculating the cube of larger integers.  This is crucial for correctness.

5. **Clearer Output:** The output messages are more informative, indicating whether the number is a cube or not.
