```java
import java.util.Arrays;

public class RemoveDuplicates {

    static int removeDuplicates(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int k = 1; // Index for the unique elements, starts at 1 since the first element is always unique initially

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[k] = nums[i];
                k++;
            }
        }

        return k; 
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 1, 2};
        int len1 = removeDuplicates(nums1);
        System.out.println("Updated length: " + len1 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums1, 0, len1))); // Output: 2, [1, 2]

        int[] nums2 = {0, 0, 1, 1, 1, 2, 2, 3, 3, 4};
        int len2 = removeDuplicates(nums2);
        System.out.println("Updated length: " + len2 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums2, 0, len2))); // Output: 5, [0, 1, 2, 3, 4]
        
        int[] nums3 = {}; // Empty array test case
        int len3 = removeDuplicates(nums3);
        System.out.println("Updated length: " + len3 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums3, 0, len3))); // Output: 0, []

        int[] nums4 = null; // Null array test case
        int len4 = removeDuplicates(nums4);
        System.out.println("Updated length: " + len4 ); // Output: 0
    }
}
```


**Explanation and Key Improvements:**

1. **Handles Edge Cases:** The code now correctly handles empty (`{}`) and `null` arrays without throwing exceptions.  It returns 0 in these cases.

2. **Clearer Variable Names:** The variable `k` (instead of `i` in some solutions) more clearly represents the index for the *unique* elements.

3. **In-place Modification:**  The `removeDuplicates` function modifies the original array `nums` directly. It overwrites the duplicate elements with the unique ones and returns the length of the updated section.

4. **Illustrative `main` Method:** The `main` method now includes multiple test cases, including edge cases and demonstrates how to print the updated array using `Arrays.copyOfRange`. This is crucial because the original array might still contain the original elements after the unique length portion.  `Arrays.copyOfRange` ensures you only print the relevant, updated part.


5. **Efficiency:** The algorithm is efficient.  It traverses the array only once (O(n) time complexity) and doesn't use extra space other than a few variables (O(1) space complexity).


This version is more robust, clear, and demonstrates the correct way to handle the results.
