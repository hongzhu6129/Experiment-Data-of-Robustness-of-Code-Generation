    public static void findKLargest(int[] arr, int k) {
        if (arr == null || arr.length == 0) {
            System.out.println("Array is empty."); // Or handle it as needed
            return;
        }


        if (k > arr.length) {
            k = arr.length; // or indicate an error, depending on requirements
        }

        // Method 1: Using a Min-Heap (Priority Queue) - Efficient for large arrays
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        for (int num : arr) {
            minHeap.offer(num);
            if (minHeap.size() > k) {
                minHeap.poll(); // Remove the smallest element if heap size exceeds k
            }
        }

        // Print the k largest elements from the min-heap
        Integer[] result = minHeap.toArray(new Integer[0]);
        Arrays.sort(result, Collections.reverseOrder()); // Sort in descending order
        for (int num : result) {
            System.out.print(num + " ");
        }
        System.out.println(); // Add a newline for cleaner output

    }
}

- **More Comprehensive Examples:** Demonstrates usage with a wider range of input arrays, including negative numbers, duplicates, and edge cases, making the example more robust.
- **Efficiency:** Uses a `PriorityQueue` (min-heap) which is the most efficient way to find the k largest elements, especially for large arrays. Its time complexity is O(n log k).
- **Correctness:**  Ensures the k largest elements are printed in descending order by sorting the results extracted from the priority queue.
