```java
public class FermatNumbers {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers:");

        for (int n = 0; n < 10; n++) {
            long fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber); 
        }
    }

    private static long calculateFermatNumber(int n) {
        return (long) Math.pow(2, Math.pow(2, n)) + 1;
    }
}
```


**Explanation and Improvements:**

1. **`calculateFermatNumber` Method:** This helper method is introduced to encapsulate the Fermat number calculation.  It takes an integer `n` as input and returns the corresponding Fermat number as a `long`.  Using a separate method improves code organization and readability.

2. **`long` Data Type:** Fermat numbers grow very rapidly.  Using `int` would quickly lead to overflow. `long` provides a larger range, but even `long` will not be able to hold all 10 Fermat numbers.  After the 5th Fermat number, you'll see incorrect values due to overflow.

3. **BigInteger (For Larger Fermat Numbers):**  To calculate and display *all* the first 10 Fermat numbers correctly, you'd need to use `BigInteger`, which can handle arbitrarily large integers:

```java
import java.math.BigInteger;

public class FermatNumbers {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers:");

        for (int n = 0; n < 10; n++) {
            BigInteger fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber);
        }
    }

    private static BigInteger calculateFermatNumber(int n) {
        return BigInteger.valueOf(2).pow((int) Math.pow(2, n)).add(BigInteger.ONE);
    }
}
```

This version using `BigInteger` will correctly calculate and display all the first 10 Fermat numbers without overflow issues.
