```java
import java.util.Scanner;

public class AverageCalculator {

    public static void main(String[] args) {

        Scanner input = new Scanner(System.in);

        System.out.print("Enter three numbers: ");

        double num1 = input.nextDouble();
        double num2 = input.nextDouble();
        double num3 = input.nextDouble();

        double average = calculateAverage(num1, num2, num3);

        System.out.println("The average is: " + average);
        input.close(); 
    }

    public static double calculateAverage(double num1, double num2, double num3) {
        return (num1 + num2 + num3) / 3;
    }
}
```

**Test Cases:**

**Normal Cases:**

1. **Input:** 10, 20, 30   **Expected Output:** 20.0
2. **Input:** 1.5, 2.5, 3.5  **Expected Output:** 2.5

7. **Input:** Double.MIN_VALUE, Double.MIN_VALUE, Double.MIN_VALUE (very small positive numbers) **Expected Output:** A very small positive number close to 0

**Improved Version (Handling Overflow/Underflow):**

```java
import java.math.BigDecimal;
import java.util.Scanner;

public class AverageCalculatorImproved {

    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        System.out.print("Enter three numbers: ");

        BigDecimal num1 = input.nextBigDecimal();
        BigDecimal num2 = input.nextBigDecimal();
        BigDecimal num3 = input.nextBigDecimal();

        BigDecimal average = calculateAverage(num1, num2, num3);

        System.out.println("The average is: " + average);
        input.close();
    }

    public static BigDecimal calculateAverage(BigDecimal num1, BigDecimal num2, BigDecimal num3) {
        BigDecimal sum = num1.add(num2).add(num3);
        return sum.divide(new BigDecimal("3"), 10, BigDecimal.ROUND_HALF_UP); // Adjust precision as needed
    }
}
```

This improved version uses `BigDecimal` to handle very large or very small numbers and avoid potential overflow/underflow issues. The `ROUND_HALF_UP` rounding mode is used for more accurate results.  You might need to adjust the precision (the second argument to `divide`) depending on the required accuracy.
