```java
import java.util.Arrays;

public class RemoveDuplicates {

    static int removeDuplicates(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int uniqueIndex = 0; // Index to track the placement of unique elements

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[uniqueIndex]) {
                uniqueIndex++; // Move to the next unique position
                nums[uniqueIndex] = nums[i]; // Place the unique element there
            }
        }

        return uniqueIndex + 1; // Return the new length (number of unique elements)
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 1, 2};
        int len1 = removeDuplicates(nums1);
        System.out.println("New length: " + len1 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums1, 0, len1))); // Output: New length: 2, Array: [1, 2]


        int[] nums2 = {0, 0, 1, 1, 1, 2, 2, 3, 3, 4};
        int len2 = removeDuplicates(nums2);
        System.out.println("New length: " + len2 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums2, 0, len2))); // Output: New length: 5, Array: [0, 1, 2, 3, 4]

        //Test cases for empty and null arrays:
        int[] nums3 = {};
        int len3 = removeDuplicates(nums3);
        System.out.println("New length: " + len3+ ", Array: " + Arrays.toString(Arrays.copyOfRange(nums3, 0, len3))); // Output: New Length: 0, Array: []


        int[] nums4 = null;
        int len4 = removeDuplicates(nums4);
        System.out.println("New length: " + len4 ); // Output: New Length: 0
    }
}
```

1. **Handles Null and Empty Inputs:** The `removeDuplicates` method now correctly handles cases where the input array is `null` or empty. It returns 0 in these scenarios.

2. **Clearer Variable Naming:** The `uniqueIndex` variable is more descriptive than just `i` or `j`, indicating its purpose.

3. **Concise Return:** The return statement is simplified to `uniqueIndex + 1`.

4. **Comprehensive Main Method:** The `main` method includes more test cases, demonstrating the function's behavior with different inputs, including edge cases like empty and null arrays.  It also uses `Arrays.copyOfRange` to clearly print only the updated portion of the array.  This prevents displaying the original, unchanged elements after the unique portion.
