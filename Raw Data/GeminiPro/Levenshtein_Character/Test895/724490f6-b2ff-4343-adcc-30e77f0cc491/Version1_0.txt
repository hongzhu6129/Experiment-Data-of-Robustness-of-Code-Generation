```java
public class RemoveElement {

    public static int removeElement(int[] nums, int val) {
        int k = 0; // Variable to track the length of the new array

        for (int i = 0; i < nums.length; i++) {
            if (nums[i] != val) {
                nums[k] = nums[i]; // Move elements not equal to val to the front
                k++;
            }
        }

        return k;
    }

    public static void main(String[] args) {
        int[] nums = {1, 4, 6, 7, 6, 2};
        int val = 6;

        System.out.print("Original array: [");
        for (int i = 0; i < nums.length; i++) {
            System.out.print(nums[i]);
            if (i < nums.length - 1) {
                System.out.print(", ");
            }
        }
        System.out.println("]");

        int newLength = removeElement(nums, val);

        System.out.print("New array (conceptually): ["); // Just to visualize - original array is modified
        for (int i = 0; i < newLength; i++) {
            System.out.print(nums[i]);
            if (i < newLength - 1) {
                System.out.print(", ");
            }
        }
        System.out.println("]");


        System.out.println("The length of the new array is: " + newLength);

       
    }
}
```


Key improvements in this version:

* **Clearer Output:** The output now clearly shows the original array and then the *conceptually* new array (represented by the first `newLength` elements of the modified original array). This helps to understand what's happening.  I've also added comments to make this distinction clear.
* **Efficiency:** The code directly modifies the input array `nums` in-place, rather than creating a new array, which is more memory-efficient.
* **Correctness:** The logic ensures that all occurrences of `val` are removed, and `k` correctly tracks the new length.
* **Adherence to Prompt:**  The code uses *only* the specified method signatures.
