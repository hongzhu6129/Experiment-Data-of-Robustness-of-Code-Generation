public class SaddlebackSearch {

    public static void main(String[] args) {
        int[][] arr = {
                {1, 4, 7, 11, 15},
                {2, 5, 8, 12, 19},
                {3, 6, 9, 16, 22},
                {10, 13, 14, 17, 24},
                {18, 21, 23, 26, 30}
        };
        int m = arr.length;
        int n = arr[0].length;

        int target = 14;
        int[] result = Saddleback(arr, m, n, target);

        if (result[0] != -1) {
            System.out.println("Element " + target + " found at row: " + result[0] + ", column: " + result[1]);
        } else {
            System.out.println("Element " + target + " not found in the array.");
        }
        
        target = 27; // Test with a not present number
        result = Saddleback(arr, m, n, target);

        if (result[0] != -1) {
            System.out.println("Element " + target + " found at row: " + result[0] + ", column: " + result[1]);
        } else {
            System.out.println("Element " + target + " not found in the array.");
        }

    }

    private static int[] Saddleback(int[][] arr, int m, int n, int target) {
        int row = 0;
        int col = n - 1;

        while (row < m && col >= 0) {
            if (arr[row][col] == target) {
                return new int[]{row, col}; // Element found
            } else if (arr[row][col] < target) {
                row++; // Move down to the next row
            } else {
                col--; // Move left to the previous column
            }
        }

        return new int[]{-1, -1}; // Element not found
    }
}

1. **Clearer Method Signature Adherence:**  The code strictly adheres to the specified method signatures: `public static void main(String[])` and `private static int[] Saddleback(int[][], int, int, int)`.

2. **Return an `int[]` for Results:** The `Saddleback` function now returns an `int[]` containing the row and column indices of the target. If the element is not found, it returns `{-1, -1}`. This makes it easier to handle the search result in `main`.

3. **Enhanced Readability:**  Variables are named more descriptively (e.g., `m`, `n`, `target`), and comments explain the logic within the `Saddleback` function.

4. **Example Usage in `main`:**  The `main` method demonstrates how to call the `Saddleback` function and interpret the returned array to determine the position of the target element or if it's not found.  It now includes an example looking for a number *not* present in the array.

5. **Efficient Saddleback Search:** The `Saddleback` function correctly implements the saddleback search algorithm, starting from the top-right corner and moving down or left based on the comparison with the target value.  This is much more efficient than a brute-force search on a sorted 2D array.
