```java
import java.util.Scanner;

public class AverageCalculator {

    public static void main(String[] args) {

        Scanner scanner = new Scanner(System.in);

        System.out.println("Enter three numbers:");

        double num1 = scanner.nextDouble();
        double num2 = scanner.nextDouble();
        double num3 = scanner.nextDouble();

        double average = calculateAverage(num1, num2, num3);

        System.out.println("The average is: " + average);
        scanner.close(); 
    }

    public static double calculateAverage(double num1, double num2, double num3) {
        return (num1 + num2 + num3) / 3;
    }
}
```

**How to Run Test Cases (Example using JUnit):**

While you could manually enter these test cases, automated testing is much more efficient. Here's how you could write a JUnit test:


```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AverageCalculatorTest {

    @Test
    void testCalculateAveragePositive() {
        assertEquals(20.0, AverageCalculator.calculateAverage(10, 20, 30));
    }

    @Test
    void testCalculateAverageNegativeAndZero() {
        assertEquals(0.0, AverageCalculator.calculateAverage(-5, 0, 5));
    }

     // ... Add tests for other cases including corner cases (large numbers, min/max values)

    @Test
    void testCalculateAverageOverflow() {
        // Check behavior with very large numbers.  May vary by Java version.
         assertThrows(ArithmeticException.class, () -> {  // Or assertEquals(Double.POSITIVE_INFINITY...
            AverageCalculator.calculateAverage(Double.MAX_VALUE, Double.MAX_VALUE, Double.MAX_VALUE);
         });
    }



}
```


Remember to add the JUnit library to your project. This automated approach makes testing much easier, especially when you make changes to your code. You can quickly rerun all your tests to ensure that everything still works correctly.  It's much better practice than manual testing.
