```java
import java.util.Arrays;

class Solution {

    private static int[] generatePrimes(int n) {
        // Create a boolean array "prime[0..n]" and initialize
        // all entries it as true. A value in prime[i] will
        // finally be false if i is Not a prime, else true.
        boolean prime[] = new boolean[n + 1];
        Arrays.fill(prime, true);

        for (int p = 2; p * p <= n; p++) {
            // If prime[p] is not changed, then it is a prime
            if (prime[p] == true) {
                // Update all multiples of p greater than or
                // equal to the square of it numbers which are
                // multiple of p and are less than p^2 are
                // already been marked.
                for (int i = p * p; i <= n; i += p)
                    prime[i] = false;
            }
        }

        // Create an array to store the prime numbers
        int primeCount = 0;
        for (int i = 2; i <= n; i++) {
            if (prime[i]) {
                primeCount++;
            }
        }

        int[] primes = new int[primeCount];
        int index = 0;
        for (int i = 2; i <= n; i++) {
            if (prime[i]) {
                primes[index++] = i;
            }
        }


        return primes;
    }


    public static void main(String[] args) {
        int n = 30; // Example: Find primes up to 30

        int[] primes = generatePrimes(n);

        System.out.print("Prime numbers up to " + n + ": ");
        for (int p : primes) {
            System.out.print(p + " ");
        }
        System.out.println();

        n = 50;  // Example: Find primes up to 50
        primes = generatePrimes(n);

        System.out.print("Prime numbers up to " + n + ": ");
        for (int p : primes) {
            System.out.print(p + " ");
        }
        System.out.println();

    }
}
```


Key improvements:

* **Efficiency:**  The Sieve of Eratosthenes is implemented correctly, marking multiples of `p` starting from `p*p` for efficiency.
* **Clarity and Readability:** Comments explain the logic, making the code easier to understand.  Variables are named meaningfully (e.g., `prime`, `primeCount`).
* **Correctness:** The code accurately finds and prints all prime numbers up to the given limit.
* **Demonstrative `main` method:** The `main` method provides examples of how to use the `generatePrimes` method with different input values.
* **Returns an array:**  The `generatePrimes` method now correctly returns an `int[]` containing the prime numbers found. This makes it more reusable.
