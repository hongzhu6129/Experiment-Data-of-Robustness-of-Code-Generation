```java
import java.util.Comparator;
import java.util.TreeMap;

public class TreeMapSortByKey {

    public static void main(String[] args) {
        // Create a TreeMap with a custom Comparator to sort keys in descending order
        TreeMap<String, Integer> treeMap = new TreeMap<>(new StringComparator());


        treeMap.put("apple", 1);
        treeMap.put("banana", 2);
        treeMap.put("orange", 3);
        treeMap.put("grape", 4);

        // Print the sorted TreeMap
        System.out.println(treeMap);


        // Example with a different Comparator (ascending order)
        TreeMap<String, Integer> ascendingTreeMap = new TreeMap<>(new Comparator<String>() {
            @Override
            public int compare(String s1, String s2) {
                return s1.compareTo(s2); // Natural ordering (ascending)
            }
        });

        ascendingTreeMap.putAll(treeMap); // Copy elements 

        System.out.println(ascendingTreeMap);


    }



    //Custom comparator class implementing Comparator<String> (must be static if nested)
    static class StringComparator implements Comparator<String> {

        @Override
        public int compare(String s1, String s2) {

            return s2.compareTo(s1); // Reverse order (descending)

        }

    }
}
```


Key improvements and explanations:

1. **Clearer Comparator Implementation:** The code now demonstrates both descending and ascending sort order using separate examples.  The `StringComparator` class implements the `Comparator<String>` interface correctly.  It's made `static` because it's a nested class and we don't want it to have an implicit reference to the enclosing `TreeMapSortByKey` instance.

2. **Demonstrates both Ascending and Descending Order:** The code shows how to create TreeMaps sorted in both ascending and descending order, making it easier to understand how Comparators control sorting.

3. **Improved Comments:**  The comments explain the code more clearly.

4. **Better Structure:** The code is structured more logically, separating the main logic from the Comparator implementation.

5. **`putAll()` for Copying:**  The code now uses `putAll()` to efficiently copy elements from one TreeMap to another when demonstrating the ascending sort order.

How to compile and run:

1. Save the code as `TreeMapSortByKey.java`.
2. Compile: `javac TreeMapSortByKey.java`
3. Run: `java TreeMapSortByKey`


This will output the TreeMap sorted first in descending order (due to `StringComparator`) and then in ascending order (due to the anonymous Comparator in the second example).
