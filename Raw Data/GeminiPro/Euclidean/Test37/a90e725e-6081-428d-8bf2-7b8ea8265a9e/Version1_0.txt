```java
import java.util.Scanner;

public class NNN {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Enter a non-zero integer n: ");
        int n = scanner.nextInt();

        if (n == 0) {
            System.out.println("Input must be a non-zero integer.");
            return; // Exit if n is zero
        }

        int nn = n * 10 + n;
        int nnn = n * 100 + nn;
        int result = n + nn + nnn;


        System.out.println("n + nn + nnn = " + result);
        scanner.close();


    }
}
```

**Corner Cases:**

* **Zero Input (0):** The code explicitly checks for this case and prints an error message because the problem statement specifies a non-zero input.
* **Large Input (e.g., 999):** The calculation will still work correctly unless the `result` exceeds the `int` data type's maximum value (2,147,483,647), in which case integer overflow will occur. Consider using `long` if you expect very large inputs.
* **Negative Input:**  The code handles negative inputs correctly.
* **Non-Integer Input:**  The `scanner.nextInt()` method will throw an `InputMismatchException` if the user enters a non-integer value.  You could add a `try-catch` block to handle this exception more gracefully if desired.  For example:
