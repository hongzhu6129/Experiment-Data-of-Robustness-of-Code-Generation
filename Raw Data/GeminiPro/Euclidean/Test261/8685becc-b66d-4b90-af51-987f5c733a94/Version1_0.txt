```java
import java.util.Arrays;

public class RemoveDuplicates {

    static int removeDuplicates(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int uniqueCount = 1; // Initialize with 1 as the first element is always unique.

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[uniqueCount] = nums[i];
                uniqueCount++;
            }
        }

        return uniqueCount;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 1, 2};
        int newLength1 = removeDuplicates(nums1);
        System.out.println("New Length: " + newLength1 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums1, 0, newLength1)));

        int[] nums2 = {0, 0, 1, 1, 1, 2, 2, 3, 3, 4};
        int newLength2 = removeDuplicates(nums2);
        System.out.println("New Length: " + newLength2 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums2, 0, newLength2)));

        // Test with an empty array
        int[] nums3 = {};
        int newLength3 = removeDuplicates(nums3);
        System.out.println("New Length: " + newLength3 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums3, 0, newLength3)));

        // Test with a null array
        int[] nums4 = null;
        int newLength4 = removeDuplicates(nums4);
        System.out.println("New Length: " + newLength4 + ", Array: " + (nums4 == null ? "null" : Arrays.toString(Arrays.copyOfRange(nums4, 0, newLength4))));
    }
}
```


**Explanation and Key Improvements:**

1. **Handles Edge Cases:**  The code explicitly handles cases where the input array is `null` or empty, returning 0 as the length.

2. **In-Place Modification:** The `removeDuplicates` function modifies the original array `nums` in place. The unique elements are moved to the beginning of the array.

3. **Clearer Variable Names:** Using `uniqueCount` instead of just `i` or `count` makes the code more readable.

4. **Efficient Logic:** The logic is streamlined and avoids unnecessary comparisons. It iterates through the array only once.

5. **Comprehensive Testing:**  The `main` method includes test cases with different arrays (including empty and null) to demonstrate the correctness of the code.

6. **Outputting the modified Array:** The code in `main` now also prints the modified array (up to the `newLength`) using `Arrays.copyOfRange`, which provides a clear view of the result.


This revised solution is more robust, efficient, and easier to understand, adhering to best practices for coding.  It directly answers the problem description's requirement of finding the updated length and also shows how the array is modified.
