```java
import java.util.Scanner;

public class AverageCalculator {

    public static void main(String[] args) {

        Scanner input = new Scanner(System.in);

        System.out.println("Enter three integers:");

        try {
            int num1 = input.nextInt();
            int num2 = input.nextInt();
            int num3 = input.nextInt();

            double average = calculateAverage(num1, num2, num3);

            System.out.println("The average is: " + average);

        } catch (java.util.InputMismatchException e) {
            System.out.println("Invalid input. Please enter integers only.");
        } finally {
            input.close(); // Close the scanner in the finally block to ensure it's closed even if an exception occurs
        }
    }

    public static double calculateAverage(int num1, int num2, int num3) {
        return (double) (num1 + num2 + num3) / 3;  // Cast to double for accurate average
    }
}
```

   - Expected Output: 20.0

2. **Negative Numbers:**
   - Input: -5 0 5
   - Expected Output: 0.0

3. **Large Numbers:**

   - Expected Output: 2.0E9 (or similar scientific notation representing 2 billion)

4. **Zeroes:**
   - Input: 0 0 0
   - Expected Output: 0.0

5. **Mixed Positive and Negative:**
   - Input: -10 20 -30
   - Expected Output: -6.666666666666667

6. **Integer Overflow (Corner Case):**
   - Input: Integer.MAX_VALUE Integer.MAX_VALUE Integer.MAX_VALUE
   - Expected Output: Will likely result in an incorrect average due to overflow. This demonstrates the limitations of using `int` for potentially very large numbers.  Consider using `long` if very large sums are expected.



7. **Input Mismatch (Corner Case):**
   - Input: 10 a 20  (Non-integer input)
   - Expected Output: "Invalid input. Please enter integers only."

Using JUnit or similar testing frameworks makes it much easier to manage and run test cases, especially as your program grows.  It helps ensure code quality and catch regressions.
