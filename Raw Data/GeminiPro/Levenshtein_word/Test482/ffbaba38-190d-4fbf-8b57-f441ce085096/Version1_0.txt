```java
import java.time.LocalDateTime;
import java.time.ZoneId;
import java.time.ZonedDateTime;
import java.time.format.DateTimeFormatter;

public class DateTimeFormats {

    public static void main(String[] args) {
        LocalDateTime now = LocalDateTime.now();
        ZonedDateTime zonedDateTime = ZonedDateTime.now();


        // yyyy-mm-dd
        System.out.println(now.format(DateTimeFormatter.ofPattern("yyyy-MM-dd")));

        // hh:mm:ss
        System.out.println(now.format(DateTimeFormatter.ofPattern("HH:mm:ss")));

        // yyyy-mm-dd hh:mm:ss
        System.out.println(now.format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));

        // e mmm yyyy hh:mm:ss.sssz  (e is day-of-week, mmm is abbreviated month)
        System.out.println(zonedDateTime.format(DateTimeFormatter.ofPattern("e MMM yyyy HH:mm:ss.SSSZ")));


        // hh:mm:ss, z (z is time zone abbreviation)
        System.out.println(zonedDateTime.format(DateTimeFormatter.ofPattern("HH:mm:ss, z")));


        // hh:mm:ss, Z (Z is time zone offset)
        System.out.println(zonedDateTime.format(DateTimeFormatter.ofPattern("HH:mm:ss, Z"))); // Using Z for zone offset


    }
}
```


Key improvements and explanations:

1. **LocalDateTime vs. ZonedDateTime:**
   - `LocalDateTime` is used for date and time without time zone information.
   - `ZonedDateTime` is used when time zone is important (as it is for several of your required formats).  We use `ZonedDateTime.now()` to get the current date and time in the system's default time zone.

2. **DateTimeFormatter:** The `DateTimeFormatter` class is the core of formatting dates and times in modern Java.  We create formatters using `DateTimeFormatter.ofPattern()` and pass the desired format string.

3. **Format Pattern Symbols:** The format pattern symbols (e.g., `yyyy`, `MM`, `dd`, `HH`, `mm`, `ss`, `z`, `Z`, `SSS`) are case-sensitive and follow the Java `DateTimeFormatter` specifications.  The code now uses the correct uppercase and lowercase symbols.

4. **Zone and Offset:**  The code now demonstrates both `z` (general time zone abbreviation) and `Z` (time zone offset like +05:30 or -0800).

5. **Error Handling (Optional but Recommended):** In a real-world application, you might want to add `try-catch` blocks to handle potential `DateTimeParseException` if an invalid format string is used.


This revised code accurately produces all the requested date and time formats, using best practices for modern Java date/time handling.
