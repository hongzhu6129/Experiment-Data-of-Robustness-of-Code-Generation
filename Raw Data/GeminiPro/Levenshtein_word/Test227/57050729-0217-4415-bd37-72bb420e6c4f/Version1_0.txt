```java
import java.util.Arrays;

class Solution {
    /**
     * Removes duplicates from a sorted array in-place and returns the new length.
     *
     * @param nums The sorted array.
     * @return The length of the array after removing duplicates.
     */
    static int removeDuplicates(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }

        int uniqueIndex = 0; // Index for placing unique elements

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[uniqueIndex]) {
                uniqueIndex++;
                nums[uniqueIndex] = nums[i];
            }
        }

        return uniqueIndex + 1;
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 1, 2};
        int len1 = removeDuplicates(nums1);
        System.out.println("Example 1: New Length = " + len1 + ", Array = " + Arrays.toString(Arrays.copyOfRange(nums1, 0, len1)));


        int[] nums2 = {0, 0, 1, 1, 1, 2, 2, 3, 3, 4};
        int len2 = removeDuplicates(nums2);
        System.out.println("Example 2: New Length = " + len2 + ", Array = " + Arrays.toString(Arrays.copyOfRange(nums2, 0, len2)));

         int[] nums3 = {};
        int len3 = removeDuplicates(nums3);
        System.out.println("Example 3: New Length = " + len3 + ", Array = " + Arrays.toString(Arrays.copyOfRange(nums3, 0, len3)));

         int[] nums4 = null;
        int len4 = removeDuplicates(nums4);
        System.out.println("Example 4: New Length = " + len4 + ", Array = null"); // No array to print in this case

    }
}
```


**Explanation and Key Improvements:**

1. **Handles Edge Cases:** The `removeDuplicates` method now correctly handles empty arrays (`nums.length == 0`) and null input (`nums == null`) by returning 0.

2. **In-Place Modification:**  The code modifies the original `nums` array directly (in-place), which is generally more efficient than creating a new array.

3. **Clearer Variable Naming:**  `uniqueIndex` is used instead of just `i` to clearly indicate its purpose.

4. **Concise Logic:** The core logic of comparing elements and moving unique elements remains efficient and easy to understand.

5. **Comprehensive Main Method:**  The `main` method now includes multiple test cases (including edge cases) to demonstrate the function's behavior and uses `Arrays.copyOfRange` to print only the relevant part of the modified array after removing duplicates.  This prevents printing the trailing garbage values that might be present in the original array.


**How it works:**

The `removeDuplicates` function uses a two-pointer approach. `uniqueIndex` keeps track of the index where the next unique element should be placed.  The loop iterates through the array, and if the current element (`nums[i]`) is different from the element at `uniqueIndex`, it's copied to the `uniqueIndex + 1` position.  Finally, `uniqueIndex + 1` is returned, representing the new length of the array containing only unique elements.
