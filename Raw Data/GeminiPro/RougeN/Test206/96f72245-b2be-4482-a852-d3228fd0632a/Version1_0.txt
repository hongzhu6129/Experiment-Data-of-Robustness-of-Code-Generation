```java
class Node {
    int data;
    Node left, right;

    public Node(int item) {
        data = item;
        left = right = null;
    }
}

public class BinaryTreeMaxDepth {

    public int maxDepth(Node node) {
        if (node == null) {
            return 0;
        } else {
            /* compute the depth of each subtree */
            int lDepth = maxDepth(node.left);
            int rDepth = maxDepth(node.right);

            /* use the larger one */
            if (lDepth > rDepth)
                return (lDepth + 1);
            else
                return (rDepth + 1);
        }
    }

    public static void main(String[] args) {
        BinaryTreeMaxDepth tree = new BinaryTreeMaxDepth();

        Node root = new Node(1);
        root.left = new Node(2);
        root.right = new Node(3);
        root.left.left = new Node(4);
        root.left.right = new Node(5);
        root.left.right.left = new Node(6); // Added a deeper node

        System.out.println("Height of tree is : " + tree.maxDepth(root));


        // Test case 2: Empty tree
        Node emptyRoot = null;
        System.out.println("Height of empty tree is : " + tree.maxDepth(emptyRoot));

        //Test case 3: Single Node tree
        Node singleNodeRoot = new Node(7);
         System.out.println("Height of single node tree is : " + tree.maxDepth(singleNodeRoot));


    }
}
```


Key improvements and explanations:

1. **Clearer Node Class:** The `Node` class is explicitly defined to represent the structure of a binary tree node.  This improves readability and organization.

2. **Recursive `maxDepth` Method:** This method implements the core logic for calculating the maximum depth using recursion.  The base case is when `node` is `null` (empty subtree), returning 0. Otherwise, it recursively calculates the depth of the left and right subtrees and returns the maximum of the two, plus 1 (for the current node).

3. **Comprehensive `main` Method:**  The `main` method now includes several test cases:
   - A sample tree with multiple levels.
   - An empty tree (to demonstrate the base case handling).
   - A tree with just a single node.  This ensures the method handles simple cases correctly.

4. **Clear Output:** The `println` statements clearly label the output, making it easy to understand the results of each test case.


This improved version is more robust, easier to understand, and demonstrates good coding practices.
