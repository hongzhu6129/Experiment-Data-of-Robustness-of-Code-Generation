


**Test Cases:**

1. **Valid Octal Number:**
   - Input: 123
   - Expected Output: 1010011

2. **Leading Zero:**
   - Input: 0123
   - Expected Output: 1010011 (Leading zeros should be handled correctly).

3. **Large Octal Number:**
   - Input: 7777777
   - Expected Output: 1111111111111111111

4. **Invalid Octal Number (containing digits other than 0-7):**
   - Input: 1283  
   - Expected Output: Invalid octal number. Digits must be between 0 and 7.

5. **Invalid Input (Non-numeric):**
   - Input: abc
   - Expected Output: Invalid input. Please enter a valid octal number.


6. **Zero:**
    - Input: 0
    - Expected Output: 0



7.  **Octal Number with internal zeros**
    - Input: 1007
    - Expected Output: 1000000111



These test cases cover various scenarios, including valid octal numbers, invalid inputs, and edge cases like leading zeros and large numbers, helping to ensure the program's correctness and robustness.
