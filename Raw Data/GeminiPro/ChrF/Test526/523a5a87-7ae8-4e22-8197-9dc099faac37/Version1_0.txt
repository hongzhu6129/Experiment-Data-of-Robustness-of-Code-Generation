```java
public class FermatNumbers {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers:");

        for (int n = 0; n < 10; n++) {
            long fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber);
        }
    }

    private static long calculateFermatNumber(int n) {
        return (long) (Math.pow(2, Math.pow(2, n))) + 1;
    }
}
```

**Explanation and Improvements:**

1. **`calculateFermatNumber(int n)` Method:**
   - This helper method encapsulates the Fermat number calculation, making the `main` method cleaner.
   - It uses `Math.pow()` to compute 2^(2^n), and then adds 1.
   - The result is cast to `long` because Fermat numbers grow very rapidly and can exceed the capacity of `int`.

2. **`main` Method:**
   - It loops from `n = 0` to `n = 9` (inclusive) to calculate the first 10 Fermat numbers.
   - It prints each Fermat number to the console.

3. **Handling Large Numbers:**
   - Even with `long`, Fermat numbers beyond the first few will exceed its capacity.  For larger values of `n`, you'd need to use `BigInteger` if you want accurate representations (see the improved version below).

**Improved Version (using BigInteger for larger numbers):**

```java
import java.math.BigInteger;

public class FermatNumbers {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers:");

        for (int n = 0; n < 10; n++) {
            BigInteger fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber);
        }
    }


    private static BigInteger calculateFermatNumber(int n) {
        BigInteger two = BigInteger.valueOf(2);
        return two.pow((int) Math.pow(2, n)).add(BigInteger.ONE); // Use BigInteger.pow
    }
}
```

This improved version uses `BigInteger` to handle arbitrarily large Fermat numbers without overflow.  It demonstrates how to work with `BigInteger`'s `pow()` and `add()` methods.
