```java
public class FermatNumbers {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers:");

        for (int n = 0; n < 10; n++) {
            long fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber); 
        }
    }

    private static long calculateFermatNumber(int n) {
        return (long) Math.pow(2, Math.pow(2, n)) + 1;
    }
}
```

**Explanation and Improvements:**

1. **`calculateFermatNumber(int n)` method:** This helper method calculates the Fermat number for a given `n`.  It uses `Math.pow()` to perform the exponentiation. The result is cast to `long` because Fermat numbers grow very quickly and can exceed the capacity of `int`.

2. **`main` method:**
   - It prints a clear header.
   - The loop iterates from `n = 0` to `n = 9` (inclusive) to calculate and display the first 10 Fermat numbers.
   - It calls `calculateFermatNumber()` within the loop to get each Fermat number.
   - The calculated Fermat number is then printed to the console.


**Limitations with `long`:**

Even using `long`, you'll encounter overflow issues fairly quickly.  Fermat numbers grow extremely rapidly. Numbers beyond the fifth Fermat number (65537) will exceed the maximum value that a `long` can hold and will result in incorrect output due to overflow.

**Handling Larger Fermat Numbers (If needed):**

If you need to work with Fermat numbers beyond the first few, you'll have to use `BigInteger` which can handle arbitrarily large integers:

```java
import java.math.BigInteger;

public class FermatNumbersBigInteger {

    public static void main(String[] args) {
        System.out.println("First 10 Fermat Numbers (using BigInteger):");

        for (int n = 0; n < 10; n++) {
            BigInteger fermatNumber = calculateFermatNumber(n);
            System.out.println(fermatNumber);
        }
    }

    private static BigInteger calculateFermatNumber(int n) {
        return BigInteger.valueOf(2).pow((int) Math.pow(2, n)).add(BigInteger.ONE);
    }
}
```

This version uses `BigInteger` for accurate calculations of larger Fermat numbers.  It's the preferred approach if you need to work with any Fermat number beyond the fifth.
