```java
public class CheckJavaInstallation {

    public static void main(String[] args) {
        System.out.println("Java Version: " + System.getProperty("java.version"));
        System.out.println("Java Runtime Version: " + System.getProperty("java.runtime.version"));
        System.out.println("Java Home: " + System.getProperty("java.home"));
        System.out.println("Java Vendor: " + System.getProperty("java.vendor"));
        System.out.println("Java Vendor URL: " + System.getProperty("java.vendor.url"));
        System.out.println("Java Class Path: " + System.getProperty("java.class.path"));

        // More robust check (checks if java command is available)
        try {
            Process p = Runtime.getRuntime().exec("java -version");
            int exitCode = p.waitFor();
            if (exitCode == 0) {
                System.out.println("\nJava is installed and accessible from the command line.");
            } else {
                System.err.println("\nJava might not be properly installed or accessible from the command line. Exit code: " + exitCode);
            }

        } catch (Exception e) {
            System.err.println("\nError checking Java installation: " + e.getMessage());
        }
    }
}
```


This improved version first prints standard Java environment details. Then, it attempts to execute the "java -version" command. A successful execution (exit code 0) confirms Java's command-line accessibility, which is crucial for many Java-related tasks.  If the command fails or an exception occurs, it indicates a potential problem with the installation.  This combined approach gives a more complete picture of the Java environment.
