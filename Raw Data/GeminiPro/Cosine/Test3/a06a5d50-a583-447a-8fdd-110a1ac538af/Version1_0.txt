```java
import java.util.Scanner;

public class MultiplicationTable {

    public static void main(String[] args) {

        Scanner input = new Scanner(System.in);

        System.out.print("Input a number: ");
        int num = input.nextInt();

        printMultiplicationTable(num);

        input.close(); 
    }

    public static void printMultiplicationTable(int num) {
        for (int i = 1; i <= 10; i++) {
            System.out.println(num + " x " + i + " = " + (num * i));
        }
    }
}
```

**Test Cases:**

1. **Positive Number:**
   - Input: 8
   - Expected Output:
     ```
     8 x 1 = 8
     8 x 2 = 16
     8 x 3 = 24
     8 x 4 = 32
     8 x 5 = 40
     8 x 6 = 48
     8 x 7 = 56
     8 x 8 = 64
     8 x 9 = 72
     8 x 10 = 80
     ```

2. **Zero:**
   - Input: 0
   - Expected Output:
     ```
     0 x 1 = 0
     0 x 2 = 0
     ...
     0 x 10 = 0
     ```

3. **Negative Number:**
   - Input: -5
   - Expected Output:
     ```
     -5 x 1 = -5
     -5 x 2 = -10
     ...
     -5 x 10 = -50
     ```

4. **Large Number:**
   - Input: 12345
   - Expected Output (first few lines):
     ```
     12345 x 1 = 12345
     12345 x 2 = 24690
     ...  // and so on up to 10
     ```


5. **Corner case: Integer.MAX_VALUE:**
   - Input: 2147483647  (Integer.MAX_VALUE)
   - Expected Output:  This will show correct multiplication for the first few lines, but eventually, integer overflow will occur, and the results will become incorrect. (This demonstrates the limitations of using `int`).  You would need to use `long` to handle larger numbers properly.  


6. **Invalid Input (Non-integer):** 
    - Input:  "abc" or "2.5" (You could test this in a separate test suite that checks for exceptions)
    - Expected behavior: The `Scanner.nextInt()` method will throw an `InputMismatchException` if the input is not a valid integer. The program should handle this gracefully (e.g., with a `try-catch` block) to avoid crashing and to provide informative error messages to the user.
