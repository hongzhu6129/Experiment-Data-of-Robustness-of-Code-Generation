```java
import java.util.Arrays;

public class RemoveDuplicates {

    static int removeDuplicates(int[] nums) {
        if (nums.length == 0) {
            return 0;
        }

        int uniqueIndex = 1; // Index of the next unique element

        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[uniqueIndex] = nums[i];
                uniqueIndex++;
            }
        }

        return uniqueIndex; 
    }

    public static void main(String[] args) {
        int[] nums1 = {1, 1, 2};
        int len1 = removeDuplicates(nums1);
        System.out.println("Updated length: " + len1 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums1, 0, len1))); // Output: 2, [1, 2]


        int[] nums2 = {0, 0, 1, 1, 1, 2, 2, 3, 3, 4};
        int len2 = removeDuplicates(nums2);
        System.out.println("Updated length: " + len2 +  ", Array: " + Arrays.toString(Arrays.copyOfRange(nums2, 0, len2))); // Output: 5, [0, 1, 2, 3, 4]

        int[] nums3 = {}; // Empty array
        int len3 = removeDuplicates(nums3);
        System.out.println("Updated length: " + len3 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums3, 0, len3))); // Output: 0, []

        int[] nums4 = {1,2,3,4,5};
        int len4 = removeDuplicates(nums4);
        System.out.println("Updated length: " + len4 + ", Array: " + Arrays.toString(Arrays.copyOfRange(nums4, 0, len4))); // Output: 5, [1, 2, 3, 4, 5]



    }
}
```

1. **Handles Empty Array:**  The code now explicitly checks for an empty input array (`nums.length == 0`) and returns 0 in that case.

2. **Efficient In-Place Modification:** The `removeDuplicates` method modifies the input array `nums` directly, avoiding the need to create a new array.  It maintains a `uniqueIndex` that tracks where the next unique element should be placed.

3. **Clearer `main` Method:** The `main` method now includes multiple test cases to demonstrate the functionality with different inputs (including an empty array and an array with no duplicates). The output is also improved to show both the updated length and the modified array (using `Arrays.copyOfRange` to print only the relevant portion).

4. **Concise and Readable:**  The code is more concise and easier to understand.  Comments explain the logic clearly.
