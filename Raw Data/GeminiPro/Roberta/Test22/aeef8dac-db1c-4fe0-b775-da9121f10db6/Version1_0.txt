```java
import java.util.Scanner;

public class OctalToHex {

    public static void main(String[] args) {

        Scanner scanner = new Scanner(System.in);
        System.out.print("Enter an octal number: ");
        String octalString = scanner.nextLine();

        try {
            // Validate input (only digits 0-7 allowed)
            for (char c : octalString.toCharArray()) {
                if (c < '0' || c > '7') {
                    throw new IllegalArgumentException("Invalid octal number: Contains characters other than 0-7.");
                }
            }

            // Octal to Decimal
            int decimal = 0;
            int power = 0;
            for (int i = octalString.length() - 1; i >= 0; i--) {
                decimal += (octalString.charAt(i) - '0') * Math.pow(8, power);
                power++;
            }


            // Decimal to Hexadecimal
            String hexString = "";
            while (decimal > 0) {
                int remainder = decimal % 16;
                char hexDigit = (remainder < 10) ? (char) (remainder + '0') : (char) (remainder - 10 + 'A');
                hexString = hexDigit + hexString;
                decimal /= 16;
            }

            if (hexString.isEmpty()) { //Handle the case of input 0
                hexString = "0";
            }

            System.out.println("Hexadecimal equivalent: " + hexString);

        } catch (NumberFormatException e) {
            System.out.println("Invalid input: Number too large."); // Handle potential overflow
        } catch (IllegalArgumentException e) {
             System.out.println(e.getMessage());  // Invalid character message
        }
    }
}
```

**Test Cases:**

1. **Valid Octal:**
   - Input: 123
   - Expected Output: Hexadecimal equivalent: 53

2. **Leading Zero:**
   - Input: 0123
   - Expected Output: Hexadecimal equivalent: 53  (Leading zeros should be ignored)

3. **Large Octal:**
   - Input: 77777777
   - Expected Output:  Hexadecimal equivalent: 7FFFFFFF (Tests handling of larger numbers)

4. **Zero:**
   - Input: 0
   - Expected Output: Hexadecimal equivalent: 0


5. **Invalid Input (containing other characters):**
   - Input: 12A3
   - Expected Output: Invalid octal number: Contains characters other than 0-7.

6. **Invalid Input (Empty String):**
    - Input: "" (just press enter)
    - Expected output:  Hexadecimal equivalent: 0 (should treat as zero)

7. **Very Large number (potential overflow):**
   - Input:  A very long string of 7s (e.g., more than 10 '7's)
   - Expected output: Invalid input: Number too large. (The program should be able to handle and report the overflow) 


These test cases cover valid octal numbers, leading zeros, large numbers, zero, and error conditions such as invalid characters and potential integer overflow due to very large inputs. They ensure the program correctly converts octal to hexadecimal and handles various input scenarios robustly.
