```java
public class OddNumbers {

    public static void printOddNumbers(int start, int end) {
        if (start < 1 || end < 1 || start > end) {
            System.out.println("Invalid input range.");
            return;
        }

        for (int i = start; i <= end; i++) {
            if (i % 2 != 0) {
                System.out.println(i);
            }
        }
    }

    public static void main(String[] args) {
        printOddNumbers(1, 99); 
    }
}
```

**Test Cases:**

1. **Normal Case:**
   - Input: `start = 1`, `end = 99`
   - Expected Output: Odd numbers from 1 to 99 printed, each on a new line (as shown in the problem description).

2. **Start greater than end:**
   - Input: `start = 10`, `end = 5`
   - Expected Output: "Invalid input range."

3. **Start less than 1:**
   - Input: `start = -5`, `end = 15`
   - Expected Output: "Invalid input range."

4. **End less than 1:**
   - Input: `start = 5`, `end = -15`
   - Expected Output: "Invalid input range."

5. **Both start and end less than 1:**
   - Input: `start = -5`, `end = -1`
   - Expected Output: "Invalid input range."

6. **Start and end are the same odd number:**
   - Input: `start = 7`, `end = 7`
   - Expected Output: `7`

7. **Start and end are the same even number:**  (Important corner case)
   - Input: `start = 8`, `end = 8`
   - Expected Output: (No output - because it's even)

8. **Range contains only even numbers:**
    - Input: `start = 2`, `end = 10`
    - Expected Output: (No output)

9. **Range starts with an even number and ends with an odd number:**
    - Input: `start = 2`, `end = 9`
    - Expected Output: 3, 5, 7, 9 (each on a new line)



These test cases cover various scenarios, including valid ranges, invalid ranges (negative numbers, start > end), and edge cases where the start and end are the same or the range contains only even numbers.  Testing these scenarios ensures that the `printOddNumbers` function is robust and handles all possible input conditions correctly.
