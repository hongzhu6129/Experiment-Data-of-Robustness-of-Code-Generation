```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class CheckJavaInstallation {

    public static boolean isJavaInstalled() {
        try {
            // Execute 'java -version' command
            Process process = Runtime.getRuntime().exec("java -version");
            BufferedReader reader = new BufferedReader(new InputStreamReader(process.getErrorStream())); // Read from error stream

            String line;
            while ((line = reader.readLine()) != null) {
                // Check if the line contains "java version" or "openjdk version"
                if (line.toLowerCase().contains("java version") || line.toLowerCase().contains("openjdk version")) {
                    return true; 
                }
            }

            //  Check exit code (0 usually means success).  Might be platform-dependent.
            int exitCode = process.waitFor();
            if(exitCode == 0) {
                 return true; //Likely Java is installed, even if version string not as expected.
            }

            return false; // Java likely not found or not executable.

        } catch (IOException | InterruptedException e) {
            return false; // Exception during execution indicates Java might not be properly installed.
        }
    }

    public static void main(String[] args) {
        if (isJavaInstalled()) {
            System.out.println("Java is installed on your computer.");
        } else {
            System.out.println("Java is NOT installed on your computer.");
        }
    }
}
```


**Test Cases:**

1. **Standard Case:**  Java is installed and accessible in the system's PATH.  Expect `true`.
2. **Java Not Installed:** Java is not present on the system at all.  Expect `false`.
3. **Java Installed but not in PATH:** Java is installed but its `bin` directory is not included in the system's PATH environment variable.  Expect `false` (unless the Java executable is in the current working directory).
4. **Java executable renamed/deleted:** Java is installed but the `java` executable has been renamed or deleted.  Expect `false`.
5. **Permissions Issue:**  Java is installed, but the user running the code doesn't have execute permissions on the `java` executable.  Expect `false`. (A more robust approach might try to detect this specific scenario.)
6. **Different Java Version String:**  Some Java distributions or embedded JVMs might have slightly different version strings.  The code checks for both "java version" and "openjdk version" to handle some of these, but there might be edge cases. Consider updating the regex if needed.
7. **Interrupted Execution:** Simulate an interruption of the `process.waitFor()` call (though this is hard to do reliably in a test). The code handles this, expecting `false`.



**How to Run Tests (Manually):**

1. **Set up different test environments:** You can either use different machines/VMs, or manipulate your system's PATH and file permissions to create the test conditions.
2. **Compile:** `javac CheckJavaInstallation.java`
3. **Run:** `java CheckJavaInstallation`
