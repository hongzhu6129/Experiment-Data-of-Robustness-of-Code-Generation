```java
import java.util.Scanner;

public class BinaryToHex {

    public static void main(String[] args) {

        Scanner input = new Scanner(System.in);
        System.out.print("Input a binary number: ");
        String binary = input.nextLine();

        try {
            int decimal = 0;
            int power = 0;
            for (int i = binary.length() - 1; i >= 0; i--) {
                if (binary.charAt(i) == '1') {
                    decimal += Math.pow(2, power);
                } else if (binary.charAt(i) != '0') {
                    System.out.println("Invalid binary number.");
                    return;
                }
                power++;
            }

            String hex = "";
            while (decimal > 0) {
                int rem = decimal % 16;
                if (rem < 10) {
                    hex = rem + hex;
                } else {
                    char hexChar = (char) ('a' + rem - 10); // Convert to a-f
                    hex = hexChar + hex;
                }
                decimal /= 16;
            }

            if(hex.isEmpty()){
                System.out.println("Hexadecimal value: '0'");
            } else{
                System.out.println("Hexadecimal value: '" + hex + "'");
            }
           

        } catch (NumberFormatException e) {
            System.out.println("Invalid binary number.");
        }

        input.close();
    }
}
```

   - Expected Output: `Invalid binary number.`

6. **Invalid Binary (contains characters other than 0 or 1):**
   - Input: `1102`
   - Expected Output: `Invalid binary number.`

7. **Invalid Binary (contains spaces):**
   - Input: `11 01`
   - Expected Output: `Invalid binary number.`

8. **Very Large Binary (potential overflow):**
    - Input: A very long string of '1's exceeding `Integer.MAX_VALUE`  (e.g., a string of 32 '1's)
    - Expected Behaviour: While the program doesn't explicitly check for integer overflow, it's good to test with large values to see how it behaves (potential overflow).  Might throw `NumberFormatException`. It's a limitation of this simple approach.

9. **Negative Binary (not directly supported):**
    - Input: `-1101`
    - Expected Output: `Invalid binary number.` (This implementation doesn't handle negative binary directly).
