


**Test Cases:**

1. **Valid Input:**
   - Input: `1101`
   - Expected Output: `d`
   - Input: `101010`
   - Expected Output: `2a`
   - Input: `11111111`
   - Expected Output: `ff`

2. **Zero:**
   - Input: `0`
   - Expected Output: `0`
   - Input: `0000` (leading zeros)
   - Expected Output: `0`

3. **Large Number:**
   - Input: `1111111111111111` (16 ones)
   - Expected Output: `ffff`

4. **Invalid Input:**
   - Input: `101201` (contains '2')
   - Expected Output: `Invalid binary input.`
   - Input: `abc`
   - Expected Output: `Invalid binary input.`
   - Input: `111 111` (contains space)
   - Expected Output: `Invalid binary input.`

5. **Empty Input:**
    - Input: "" (Empty String)
    - Expected Output: "0"



This improved version provides more robust handling of invalid input and corner cases like leading zeros, empty strings, and larger binary numbers. It avoids using any methods outside of `main` as requested.  The inclusion of clear error messages helps in debugging and understanding if the input is not valid.
