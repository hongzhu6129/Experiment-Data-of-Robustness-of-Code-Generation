```java
import java.util.Scanner;

public class ProductOfTwoNumbers {

    public static void main(String[] args) {

        Scanner scanner = new Scanner(System.in);

        System.out.print("Enter the first number: ");
        long num1 = scanner.nextLong(); // Use long to handle larger numbers

        System.out.print("Enter the second number: ");
        long num2 = scanner.nextLong();

        long product = num1 * num2;

        System.out.println("The product of " + num1 + " and " + num2 + " is: " + product);
        scanner.close();
    }
}
```

**Explanation of Corner Cases and Overflow:**

* **Test Case 8 & 10:**  `Long.MAX_VALUE * 2` and `Long.MIN_VALUE * 2` will result in overflow. Java doesn't throw an exception for integer overflow; instead, it wraps around. This is why you get unexpected negative or zero results.

**How to Improve Overflow Handling (Optional):**

If you need to detect and handle overflow explicitly, you could use the `Math.multiplyExact()` method introduced in Java 8.  This method throws an `ArithmeticException` if the result overflows.

}
