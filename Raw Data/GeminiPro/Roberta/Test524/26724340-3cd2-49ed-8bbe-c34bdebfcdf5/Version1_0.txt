```java
public class CubeChecker {

        if (isCube(num)) {

        } else {

        testCube(27);  // Cube

        testCube(-8); // Cube (of -2)
        testCube(0); // Cube (of 0)
        testCube(1); // Cube (of 1)
        testCube(1000); // Cube of 10

        if(args.length > 0) {

                int userNum = Integer.parseInt(args[0]);
                testCube(userNum);

            } catch (NumberFormatException e) {
                System.out.println("Invalid input. Please enter an integer.");
            }
        }

    }

    private static boolean isCube(int num) {
        // Efficiently handle cubes of negative numbers
        int absNum = Math.abs(num);

        // Binary search for the cube root (efficient for large numbers)
        int low = 0;
        int high = absNum;  // Highest possible cube root is less than the number itself

        while(low <= high) {
             int mid = low + (high - low)/2;
             int cube = mid * mid * mid;
             if(cube == absNum) {

                // Check the original sign of the number.
                if(num < 0) {
                    return mid * mid * mid * -1 == num; // Negative numbers
                }

                return true;
             }
             else if(cube < absNum) {
                 low = mid + 1;
             }
             else{
                 high = mid - 1;
             }
         }
         return false;
    }




    private static void testCube(int num) {
        if (isCube(num)) {
            System.out.println(num + " is a cube.");
        } else {
            System.out.println(num + " is not a cube.");
        }
    }
}
```


Key improvements and explanations:

* **Handles negative cubes:** The `isCube` method now correctly handles negative numbers.  We take the absolute value for the cube root calculation and then check if the cube of the potential root *matches the original sign* of the input `num`.
* **Efficiency:** The code now uses a binary search approach within `isCube`. This is significantly more efficient than a brute-force loop or using `Math.cbrt()` when dealing with larger numbers.
* **Clearer Testing:** The `testCube` helper method makes it easier to test with multiple input values.
* **User Input:** The code now takes user input from command line arguments. This makes it more interactive. It also includes error handling to ensure that the input is a valid integer.
