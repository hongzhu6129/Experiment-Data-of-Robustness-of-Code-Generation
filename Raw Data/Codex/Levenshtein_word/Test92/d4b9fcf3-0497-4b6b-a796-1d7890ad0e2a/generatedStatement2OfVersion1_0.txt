

**Test Cases:**

Here are some test cases covering various scenarios, including corner cases:

1. **Normal Case:**
   - Input: 10, 20, 30
   - Expected Output: 20.0

2. **Negative Numbers:**
   - Input: -5, -10, -15
   - Expected Output: -10.0

3. **Zero Values:**
   - Input: 0, 0, 0
   - Expected Output: 0.0

4. **Mixed Positive and Negative:**
   - Input: -5, 10, 15
   - Expected Output: 6.666666666666667

5. **Large Numbers:**
   - Input: 1000000, 2000000, 3000000
   - Expected Output: 2000000.0


6. **Decimal Numbers:**
   - Input: 2.5, 3.7, 4.1
   - Expected Output: 3.4333333333333336

7. **Corner Case - Very Large Numbers (potential overflow with integers, but handled correctly with doubles):**
    - Input:  Double.MAX_VALUE, Double.MAX_VALUE/2,  Double.MAX_VALUE/4  
    - Expected Output: A very large number (approximately 1.44E308, may vary slightly depending on the system)



8. **Corner Case: Infinity:**
   - Input: Double.MAX_VALUE, Double.MAX_VALUE, Double.MAX_VALUE
   - Expected Output: Infinity


9. **Invalid Input - Non-Numeric:**
   - Input: a, b, c
   - Expected Output: "Invalid input. Please enter numeric values."
