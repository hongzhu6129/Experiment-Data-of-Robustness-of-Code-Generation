


Key improvements:

- **Handles leap years correctly:** Uses `YearMonth.lengthOfMonth()` to accurately get the number of days, accounting for leap years.
- **Clearer output:**  Uses `getDisplayName()` for a more user-friendly month name.
- **Handles a given month:**  Demonstrates how to work with a specific `Month` and `year`.
- **Error Handling:** Includes error checking for the command-line arguments, handling `NumberFormatException` and invalid month values.
- **Command-line arguments:** Shows how to get the year and month from command-line arguments to make the program more flexible.



How to compile and run with arguments:

1. **Save:** Save the code as `GetMonthInfo.java`.
2. **Compile:** `javac GetMonthInfo.java`
3. **Run:**  `java GetMonthInfo 2025 3` (This would give info for March 2025. Replace `2025` and `3` with desired year and month (1-12). ) Run without arguments to get the current month and the example (February 2024).
