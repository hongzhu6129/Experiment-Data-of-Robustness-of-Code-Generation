


**Test Cases:**

**1. Positive Cases (Two or more have the same rightmost digit):**

* **Input:** 22, 32, 45   **Expected Output:** true
* **Input:** 10, 20, 30   **Expected Output:** true
* **Input:** 11, 11, 11   **Expected Output:** true
* **Input:** 7, 17, 207  **Expected Output:** true
* **Input:** 100, 200, 100   **Expected Output:** true


**2. Negative Cases (No two numbers have the same rightmost digit):**

* **Input:** 1, 2, 3   **Expected Output:** false
* **Input:** 12, 23, 34   **Expected Output:** false
* **Input:** 10, 21, 32   **Expected Output:** false


**3. Corner Cases:**

* **All zeros:**  **Input:** 0, 0, 0   **Expected Output:** true
* **Two zeros and a different number:** **Input:** 0, 0, 5   **Expected Output:** true
* **Large Numbers:** **Input:** 12345, 67895, 10005   **Expected Output:** true
* **Numbers with leading zeros (treated as the number without zeros):** **Input:** 001, 011, 21  **Expected Output:** true  (because 1, 11, and 21 are considered)

**4.  Negative Input (Error Handling):**

* **Input:** -1, 2, 3  **Expected Output:** "Input must be non-negative integers."
* **Input:** 1, -2, 3  **Expected Output:** "Input must be non-negative integers."



This set of test cases covers a good range of scenarios, including normal cases, edge cases where all or some inputs are zero,  large number inputs, and handling of potentially invalid (negative) input.  Remember to test your code with these cases to ensure its correctness.
