

**Test Cases:**

Here's a set of test cases covering various scenarios, including corner cases:

| Test Case | Input 1 | Input 2 | Input 3 | Expected Output |
|---|---|---|---|---|
| Positive Numbers | 10 | 20 | 30 | 20.0 |
| Negative Numbers | -10 | -20 | -30 | -20.0 |
| Mixed Numbers | -5 | 15 | 25 | 11.666666666666666 |
| Zero Values | 0 | 0 | 0 | 0.0 |
| Large Numbers | 1000000 | 2000000 | 3000000 | 2000000.0 |
| Small Numbers | 0.001 | 0.002 | 0.003 | 0.002 |
| Decimal Numbers | 2.5 | 5.7 | 8.9 | 5.7 |
| Same Numbers | 7 | 7 | 7 | 7.0 |
| One Large, Two Small | 1000000 | 1 | 2 | 333334.3333333333 |



**Explanation of Corner Cases:**

* **Zero Values:**  Testing with all zeros ensures the program handles this basic case correctly.
* **Large Numbers:**  Using very large numbers helps check for potential overflow issues (though `double` has a wide range).
* **Small Numbers:** Using very small numbers checks for potential underflow or precision issues.
* **One Large, Two Small:**  This case checks if the average calculation is skewed significantly by one dominant number, ensuring proper weighting.



**How to run the test cases (Manually):**

1. Compile the Java code: `javac AverageCalculator.java`
2. Run the compiled code: `java AverageCalculator`
3. Enter the input values for each test case when prompted, and verify the output against the expected output.
