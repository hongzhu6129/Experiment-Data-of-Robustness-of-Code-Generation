



**Test Cases:**

* **Valid Hexadecimal Numbers:**
    * `0`:  Expected output: `0000`
    * `1`:  Expected output: `0001`
    * `9`:  Expected output: `1001`
    * `A`:  Expected output: `1010`
    * `a`:  Expected output: `1010` (case-insensitive)
    * `F`:  Expected output: `1111`
    * `f`:  Expected output: `1111` (case-insensitive)
    * `1A`: Expected output: `00011010`
    * `FF`: Expected output: `11111111`
    * `Cafe`: Expected output: `1100101011111110` (case-insensitive)
    *  `123456789ABCDEF`:  (A longer test case)


* **Corner Cases:**
    * **Empty Input:** ""  Expected behavior: Print "Please provide a hexadecimal number..." and exit.
    * **Invalid Characters:** `G`, `H`, `Z`, `!`, `@`, `#`, etc. Expected behavior: Print "Invalid hexadecimal character: [character]" and exit.
    * **Mixed Case:** `aBcDeF`  Expected output: `101010111100110111101111` (handles mixed case correctly)
    * **Leading Zeroes:** `000A` Expected output: `0000000000001010` (handles leading zeroes)



**How to run the tests:**

1. Save the code as `HexToBinary.java`.
2. Compile: `javac HexToBinary.java`
3. Run with a test case: `java HexToBinary FF` (replace `FF` with your test case).  You can also omit the argument to test the empty input case.


By testing with these different scenarios, you can verify that the program correctly converts hexadecimal numbers to binary and handles invalid input appropriately.
