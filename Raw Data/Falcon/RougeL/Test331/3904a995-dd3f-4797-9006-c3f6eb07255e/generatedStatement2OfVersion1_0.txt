


**Test Cases:**

**1. Valid Rectangle:**
Input: 
Shape: rectangle
Length: 5
Width: 3
Expected Output: 15.0

**2. Valid Square:**
Input:
Shape: square
Side: 4
Expected Output: 16.0

**3. Valid Triangle:**
Input:
Shape: triangle
Base: 6
Height: 4
Expected Output: 12.0

**4. Invalid Shape:**
Input:
Shape: circle
Expected Output: Invalid shape entered.

**5. Zero Length Rectangle:**
Input:
Shape: rectangle
Length: 0
Width: 5
Expected Output: Invalid input: Length and width must be positive.

**6. Negative Width Rectangle:**
Input:
Shape: rectangle
Length: 5
Width: -2
Expected Output: Invalid input: Length and width must be positive.

**7. Zero Side Square:**
Input:
Shape: square
Side: 0
Expected Output: Invalid input: Side length must be positive.


**8. Negative Base Triangle:**
Input:
Shape: triangle
Base: -3
Height: 4
Expected Output: Invalid input: Base and height must be positive.


**9. Case Insensitive Shape Input:**
Input:
Shape: ReCtAnGlE 
Length: 2
Width: 7
Expected Output: 14.0


These test cases cover valid inputs for different shapes, an invalid shape, and corner cases where dimensions are zero or negative. They ensure that the program handles both expected and unexpected inputs correctly.  Using `scanner.nextLine().toLowerCase()` also ensures the shape input is case-insensitive, making the program more robust.
