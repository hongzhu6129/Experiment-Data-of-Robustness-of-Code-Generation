


**Test Cases:**

**1. All same rightmost digit:**

* Input: 11, 21, 31
* Expected Output: true

**2. Two same, one different:**

* Input: 12, 22, 33
* Expected Output: true

**3. All different rightmost digits:**

* Input: 11, 22, 33
* Expected Output: false

**4. Zeroes:**

* Input: 10, 20, 30   // All end in 0
* Expected Output: true

* Input: 10, 20, 31 // Two end in 0
* Expected Output: true

* Input: 10, 21, 32 // One ends in 0
* Expected Output: false


**5. Single-digit numbers:**

* Input: 1, 1, 2
* Expected Output: true

**6. Large Numbers:**

* Input: 12345, 67895, 100005  // Testing with larger numbers
* Expected Output: true

**7. Corner Cases (boundary conditions):**

* Input: 0, 0, 0   // All zeros
* Expected Output: true

* Input: Integer.MAX_VALUE, Integer.MAX_VALUE - 1, Integer.MAX_VALUE -10 // Very large numbers
* Expected Output:  Will depend on the rightmost digit of Integer.MAX_VALUE (which varies with Java implementation - it's usually 7)


**8. Negative Input (Error Handling):**

* Input: -1, 2, 3
* Expected Output: "Please enter non-negative integers." (or similar error message)



These test cases cover a good range of scenarios, including typical cases, cases with zeros, single-digit numbers, large numbers, and important boundary conditions.  Testing negative input is crucial to verify your error handling works as expected.
