


**Test Cases:**

Here's a set of test cases covering various scenarios, including corner cases:

1. **Positive Numbers:**
   - Input: 10, 20, 30
   - Expected Output: 20.0

2. **Negative Numbers:**
   - Input: -5, -10, -15
   - Expected Output: -10.0

3. **Mixed Positive and Negative Numbers:**
   - Input: -5, 10, 20
   - Expected Output: 8.333333333333334

4. **Zero Values:**
   - Input: 0, 0, 0
   - Expected Output: 0.0

5. **Large Numbers:**
   - Input: 1000000, 2000000, 3000000
   - Expected Output: 2000000.0

6. **Decimal Numbers:**
   - Input: 2.5, 5.7, 8.9
   - Expected Output: 5.7

7. **Corner Case: Very Large Numbers (potential overflow if using integers):**
   - Input:  Double.MAX_VALUE, Double.MAX_VALUE, Double.MAX_VALUE / 2   // Handling potential overflow with doubles
   - Expected Output: Double.POSITIVE_INFINITY (or a very large number)

8. **Corner Case: Very Small Numbers (close to zero):**
    - Input: 0.0000001, 0.0000002, 0.0000003
    - Expected Output: 2.0E-7 (or a very small number close to 0)
