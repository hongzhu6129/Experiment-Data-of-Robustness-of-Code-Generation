


**Test Cases:**

1. **Valid Octal Number:**
   - Input: 123
   - Expected Output: Hexadecimal equivalent: 53

2. **Octal Number with Leading Zeroes:**
   - Input: 00077
   - Expected Output: Hexadecimal equivalent: 3F

3. **Octal Number with a large value:**
   - Input: 7777777
   - Expected Output:  Hexadecimal equivalent: 7FFFFF

4. **Invalid Octal Number (Contains digits other than 0-7):**
   - Input: 1289
   - Expected Output: Invalid input: Invalid octal digit: 8

5. **Invalid Octal Number (Empty String):**
    - Input: "" (just press enter)
    - Expected Output: Invalid input: For input string: ""   (This specific message depends on your Java version, but it should indicate invalid input).

6. **Invalid Octal Number (Negative sign):**
    - Input: -123
    - Expected Output: Invalid input: Invalid octal digit: -

7. **Corner Case: Zero:**
    - Input: 0
    - Expected Output: Hexadecimal equivalent: 0

8. **Corner Case: Single Digit:**
     - Input: 7
     - Expected Output: Hexadecimal equivalent: 7


These test cases cover various scenarios, including valid octal inputs, invalid inputs (containing invalid digits, empty string, negative sign), and corner cases like zero and single-digit octal numbers.  Running your code with these examples will help ensure that your octal-to-hexadecimal conversion works correctly.
