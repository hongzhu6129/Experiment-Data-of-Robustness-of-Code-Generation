


**Test Cases:**

1. **Valid Octal Number:**
   - Input: 123
   - Expected Output: Hexadecimal equivalent: 53

2. **Octal Number with Leading Zeroes:**
   - Input: 00077
   - Expected Output: Hexadecimal equivalent: 3F


3. **Octal Number with a large value:**
   - Input: 7777777
   - Expected Output: Hexadecimal equivalent: FFFFFF


4. **Invalid Octal Number (Contains digits other than 0-7):**
   - Input: 1283 
   - Expected Output: Invalid octal number: Invalid octal digit: 8

5. **Invalid Octal Number (Empty String):**
   - Input: "" (Just press enter)
   - Expected Output: Hexadecimal equivalent: 0  (Handles this gracefully). Note, it will be 0 unlike some other conversions.


6. **Invalid Octal Number (Negative Number):**  Though not strictly octal, good to test.
   - Input: -123
   - Expected Output: Invalid octal number: Invalid octal digit: -

7. **Zero:**
   - Input: 0
   - Expected Output: Hexadecimal equivalent: 0

8. **Single Digit:**
   - Input: 7
   - Expected Output: Hexadecimal equivalent: 7



These test cases cover various scenarios, including valid octal numbers, invalid input (containing invalid digits, empty input), and edge cases like zero and leading zeros. They help to verify that the code handles different situations correctly and produces the expected output.
