



**Test Cases:**

**Normal Cases:**

1. **Input:** 22, 32, 45  **Expected Output:** true (22 and 32 have the same rightmost digit)
2. **Input:** 10, 20, 30  **Expected Output:** true (All have 0 as the rightmost digit)
3. **Input:** 1, 2, 3   **Expected Output:** false
4. **Input:** 123, 456, 789 **Expected Output:** false
5. **Input:** 11, 12, 13 **Expected Output:** true (first number shares with others)
6. **Input:** 10, 21, 31 **Expected Output:** true


**Corner Cases:**

1. **Input:** 0, 0, 0   **Expected Output:** true (All zeroes)
2. **Input:** 100, 200, 300 **Expected Output:** true (Multiples of 100).
3. **Input:** 1, 1, 1   **Expected Output:** true (All the same number)
4. **Input:** 1, 11, 21 **Expected Output:** true
5. **Large Numbers:** 123456789, 987654321, 111111111 **Expected Output:** true (Checks if the logic handles large numbers correctly -  987654321 and 111111111 end in 1)



**Negative Input (Error Handling Test):**

1. **Input:** -1, 2, 3  **Expected Output:** "Invalid input: Integers must be non-negative" (or similar error message)


These test cases cover various scenarios, including situations where two or three integers have the same rightmost digit, cases with zeroes, and inputs involving larger integers.  The negative input test verifies that your error handling works as expected.  It's crucial to test your code with a diverse range of inputs to ensure its correctness and robustness.
