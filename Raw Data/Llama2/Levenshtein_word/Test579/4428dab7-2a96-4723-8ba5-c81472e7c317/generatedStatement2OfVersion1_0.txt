



**Test Cases:**

1. **Valid Binary:**
   - Input: `1101`
   - Expected Output: `Hexadecimal value: 'd'`

2. **Leading Zeros:**
   - Input: `0001111`
   - Expected Output: `Hexadecimal value: 'f'`

3. **Zero:**
   - Input: `0`
   - Expected Output: `Hexadecimal value: '0'`

4. **Large Binary Number:**
   - Input: `1111111111111111` (16 ones)
   - Expected Output: `Hexadecimal value: 'ffff'`

5. **Empty Input:**
   - Input: ``
   - Expected Output: `Hexadecimal value: '0'`

6. **Invalid Input (contains characters other than 0 or 1):**
   - Input: `101a01`
   - Expected Output: `Invalid binary input.`

7. **Invalid Input (Space):**
   - Input: "101 011"
   - Expected Output: `Invalid binary input.`


These test cases cover various scenarios including valid binary numbers, handling leading zeros, the special case of zero, a large binary number to test potential overflow issues (although long should handle 16 bits comfortably), empty input, and invalid input containing non-binary characters and spaces to ensure robust error handling.
