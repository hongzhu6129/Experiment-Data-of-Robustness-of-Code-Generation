



**Test Cases:**

**1. Positive Case (Java Installed):**

* **Expected Outcome:**  The program prints "Java is installed on this system."
* **How to Test:** Run the program on a system with a standard Java installation (e.g., Oracle JDK, OpenJDK).

**2. Negative Case (Java Not Installed):**

* **Expected Outcome:** The program prints "Java is NOT installed on this system."
* **How to Test:**
    * Uninstall Java if you have it.  Then run the program.
    * Or, run the program in a Docker container or virtual machine that doesn't have Java installed.

**3. Corner Case 1 (Java in Non-Standard Location):**

* **Expected Outcome:**  May print "Java is NOT installed".
* **How to Test:**  If Java is installed but not in the system's PATH environment variable, the `java -version` command may fail.  Modify your system's PATH to *temporarily* remove the Java directory to simulate this condition.

**4. Corner Case 2 (Java Executable Renamed):**

* **Expected Outcome:** Prints "Java is NOT installed".
* **How to Test:** (Advanced/Risky - Avoid on production systems) If possible, *temporarily* rename the `java` executable in your Java installation directory (e.g., to `java_renamed`). This tests the reliance on the specific executable name.

**5. Corner Case 3 (Permissions Issues):**

* **Expected Outcome:**  May print "Java is NOT installed".
* **How to Test:** (Advanced/Risky - Avoid on production systems) Temporarily change the file permissions on the `java` executable to prevent execution.  This simulates a situation where Java is present but the user doesn't have the right to execute it.


**Important Notes for Testing:**

* **Restore your system:** After testing corner cases 3, 4, and 5, remember to restore your system to its original state.  Do not leave these modifications in place, as they can break other Java applications.
* **Alternative checks (Advanced):**  For more robust detection (especially handling corner cases), you might consider looking into the registry (on Windows) or using the `which` command (on Linux/macOS) to determine the Java installation path.  However, these methods are platform-specific.
