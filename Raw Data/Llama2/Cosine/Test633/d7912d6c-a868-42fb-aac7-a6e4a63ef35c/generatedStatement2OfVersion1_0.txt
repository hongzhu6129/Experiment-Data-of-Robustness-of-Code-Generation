

**Test Cases:**

Here's a set of test cases covering various scenarios, including corner cases:

1. **Positive Numbers:**
   - Input: 10, 20, 30
   - Expected Output: 20.0

2. **Negative Numbers:**
   - Input: -5, -10, -15
   - Expected Output: -10.0

3. **Mixed Positive and Negative:**
   - Input: -5, 10, 25
   - Expected Output: 10.0

4. **Zeroes:**
   - Input: 0, 0, 0
   - Expected Output: 0.0

5. **Large Numbers:**
   - Input: 1000000, 2000000, 3000000
   - Expected Output: 2000000.0

6. **Decimal Numbers:**
   - Input: 2.5, 5.5, 7.5
   - Expected Output: 5.166666666666667

7. **Corner Cases:**
   - **Integer Overflow/Underflow (less likely with doubles but still possible with extremely large magnitudes):**  Test with `Double.MAX_VALUE`, `Double.MIN_VALUE`, and combinations that might lead to exceeding these limits (e.g.,  `Double.MAX_VALUE`, `Double.MAX_VALUE`, `Double.MAX_VALUE`). The program should ideally handle these gracefully and not crash.  Expect `Infinity` or `-Infinity` as a possible output in overflow/underflow situations.
   - **Invalid Input:**  "abc", "1, 2, 3" (commas instead of spaces), "1.5.5" (invalid double format). The program should handle these gracefully, displaying an error message and not crashing.
