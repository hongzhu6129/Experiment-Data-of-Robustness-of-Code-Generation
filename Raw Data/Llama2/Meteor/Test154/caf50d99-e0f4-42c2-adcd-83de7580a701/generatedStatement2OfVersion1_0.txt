

**Test Cases:**

Here's a set of test cases covering various scenarios, including corner cases:

1. **Original Example:**
   - Latitude 1: 25
   - Longitude 1: 35
   - Latitude 2: 35.5
   - Longitude 2: 25.5
   - Expected Distance: ~1480.08 km

2. **Same Point:**
   - Latitude 1: 40
   - Longitude 1: -70
   - Latitude 2: 40
   - Longitude 2: -70
   - Expected Distance: 0 km

3. **North and South Poles:**
   - Latitude 1: 90  (North Pole)
   - Longitude 1: 0
   - Latitude 2: -90 (South Pole)
   - Longitude 2: 0 
   - Expected Distance: ~20015.08 km (approximately Earth's circumference / 2)

4. **Antipodes (opposite sides of Earth):**
   - Latitude 1: 40
   - Longitude 1: -70
   - Latitude 2: -40
   - Longitude 2: 110
   - Expected Distance: ~20015.08 km  (approximately Earth's circumference / 2)


5. **Points on the Equator:**
   - Latitude 1: 0
   - Longitude 1: 0
   - Latitude 2: 0
   - Longitude 2: 90
   - Expected Distance: ~10007.54 km (approximately Earth's circumference / 4)


6. **Points Near the Date Line (Longitude 180/-180):**
   - Latitude 1: 10
   - Longitude 1: 179
   - Latitude 2: 10
   - Longitude 2: -179
   - Expected Distance:  A small value (because they are close despite the longitude difference)


7. **Invalid Input (Optional - depends on how you handle it):**
   - Latitude 1: "abc"  (or any non-numeric input)
   - ... (other inputs)
   - Expected:  An error message or exception. Your program should handle this gracefully.



**How to Test:**

1. **Manual Testing:**  Run the program and enter the input values for each test case. Compare the output distance with the expected distance.

2. **Automated Testing (Recommended):** Use a testing framework like JUnit. Create test methods for each test case. Use `assertEquals()` (with a delta for floating-point comparisons) to check if the calculated distance is close enough to the expected distance.  This approach makes testing more thorough and repeatable.
