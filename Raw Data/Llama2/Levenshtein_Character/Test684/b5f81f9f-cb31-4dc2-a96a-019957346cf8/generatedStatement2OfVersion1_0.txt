6. **Invalid Octal Number (Non-numeric characters):**
   - Input: 12A3
   - Expected Output: Invalid octal number.

   - Input: 
   - Expected Output: Invalid input. Please enter a valid octal number. (This depends on how the scanner handles empty input, it might throw an exception or read the next line).

    - Input: -123 
    - Expected Output: Invalid octal number.


These test cases cover various scenarios, including valid octal inputs, invalid inputs (containing invalid digits, non-numeric characters, negative numbers) zero, large numbers, and leading zeroes, helping to ensure that the program functions correctly under different conditions.
