



**Test Cases:**

**1. Positive Cases:**

* **Input:** 22, 32, 45  **Expected Output:** true (22 and 32 share the rightmost digit 2)
* **Input:** 10, 20, 30  **Expected Output:** true (All share the rightmost digit 0)
* **Input:** 107, 207, 306 **Expected Output:** true (107 and 207 share the rightmost digit 7)
* **Input:** 1, 11, 111 **Expected Output:** true



**2. Negative Cases:**

* **Input:** 1, 2, 3  **Expected Output:** false
* **Input:** 12, 23, 34  **Expected Output:** false
* **Input:** 101, 102, 103 **Expected Output:** false



**3. Corner/Edge Cases:**

* **Input:** 0, 0, 0  **Expected Output:** true (All share the rightmost digit 0)
* **Input:** 0, 10, 20 **Expected Output:** true
* **Input:** 1, 1, 1 **Expected Output:** true
* **Input:**  Integer.MAX_VALUE, Integer.MAX_VALUE, Integer.MAX_VALUE -1  **Expected Output:** true (Tests large numbers and potential overflow issues, though the modulo operator handles this well).
* **Input:** 123456789, 987654321, 111111119 **Expected Output:** true (Tests longer numbers)



**How to Run Test Cases:**

1.  **Manually:** You can run these test cases by entering the input values when prompted by the program.
2.  **Automated Testing (Recommended):** Using a testing framework like JUnit is the best approach. You can write test methods that call the `function` with different inputs and assert that the output matches your expected results. This makes testing more efficient and repeatable.
