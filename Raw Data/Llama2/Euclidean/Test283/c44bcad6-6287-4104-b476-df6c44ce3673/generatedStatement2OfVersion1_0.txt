

**Test Cases:**

Here are some test cases covering normal scenarios, edge cases, and corner cases:

**1. Normal Cases:**

* **Input:** 10, 20, 30   **Expected Output:** 20.0
* **Input:** 2.5, 5.5, 8.0   **Expected Output:** 5.333333333333333
* **Input:** -5, 0, 5  **Expected Output:** 0.0

**2. Edge Cases:**

* **Input:** 0, 0, 0   **Expected Output:** 0.0  (All zeros)
* **Input:** Integer.MAX_VALUE, Integer.MAX_VALUE, Integer.MAX_VALUE  **Expected Output:** 2.147483647E9 (Handling large values - potential overflow if using `int` instead of `double`)

* **Input:** 1, 2, a   **Expected Behavior:** `InputMismatchException` (Non-numeric input) � The provided code will throw this exception.  Robust code should include `try-catch` blocks to handle this gracefully.
* **Input:** (Empty input) **Expected Behavior:**  `NoSuchElementException` � Robust code should prompt the user again or handle the absence of input.
* **Input:** Integer.MAX_VALUE, Integer.MAX_VALUE, Integer.MIN_VALUE **Expected Output:** 7.15827882E8 (Handling a mix of extremely large positive and negative values).  Test for potential overflow/underflow.
* **Input:** Double.MAX_VALUE, Double.MAX_VALUE, Double.MAX_VALUE  **Expected Output:** Infinity (Handling Double.MAX_VALUE � expect Infinity as result)
* **Input:** Double.NaN, 1, 2 **Expected Output:** NaN (Handling Not a Number � the average will be NaN)
